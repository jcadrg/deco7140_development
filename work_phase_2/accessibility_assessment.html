Accessibility Assessment

This section outlines how this website adheres to WCAG 2.1 accessibility guidelines to ensure an inclusive user experience.

Principle 1: Perceivable

  • 1.1.1 Alternative Text: All non-text content includes descriptive alt attributes (e.g., icons, images).
  • 1.3.2 Meaningful Sequence: HTML structure uses semantic tags to ensure the reading order matches the visual layout.
  • 1.4.1 Use of Color: Information is never conveyed by color alone; text and icons are also used.
  • 1.4.3 Contrast: Sufficient contrast ratios exist between text and background colors to ensure readability.
  • 1.4.4 Resize Text: Text remains legible when zoomed up to 200% without loss of content or functionality.

Principle 2: Operable

  • 2.1.1 Keyboard Navigation: All interactive elements (like menus and cards) are accessible using a keyboard alone.
  • 2.4.7 Focus Visible: Focus indicators are preserved and clearly visible to support keyboard users.

Principle 3: Understandable

  • 3.1.5 Reading Level: All written content is written in plain language to accommodate users of varying literacy levels.
  • 3.2.3 Consistent Navigation: Navigation remains in the same order and style across all pages for predictability.
  • 3.2.4 Consistent Identification: Reusable UI components (buttons, cards, menus) maintain a consistent appearance and behavior.

Principle 4: Robust

  • 3.3.1 Error Identification: Any user input forms (e.g., future contact forms) will clearly display errors and offer guidance.
  • 3.3.2 Labels and Instructions: All form fields include clear, descriptive labels and placeholder text.
  • 3.3.4 Error Prevention: Where applicable, input validation and confirmation will be used to prevent mistakes and assist with recovery.
